Java — Strings Medium

Q1: Maximum Substring

You are given a string s. A substring is any contiguous sequence of characters from the string.

Your task is to:
  • Consider all possible substrings of the string.
  • Determine which substring would appear last if all substrings were sorted alphabetically.
  • Return that substring.
Example 1: s = "baca"
All unique substrings: ["b","ba","bac","baca","a","ac","aca","c","ca"]
Arranged alphabetically: ["a","ac","aca","b","ba","bac","baca","c","ca"]
The alphabetically maximum substring is "ca".

Example 2: s = "aaa" → Answer: "aaa"

Constraints: 1 ≤ length of s ≤ 100. All characters are lowercase a-z.
Explain — What is this question actually asking? ▶

In simple words: A "substring" is any piece you can cut from the string by taking consecutive characters. From "baca" you can cut: "b", "ba", "bac", "baca", "a", "ac", "aca", "c", "ca".

"Alphabetically maximum" means: if you put all these pieces in dictionary order, which one comes LAST? Just like "zebra" comes after "apple" in a dictionary, "ca" comes after "baca" because 'c' > 'b'.

What you need to do: Find the substring that comes LAST in dictionary order. You don't need to actually sort all substrings — there's a smarter way.

Phase 1 — Analyze the question type ▶

Type: String manipulation + alphabetical comparison

Concepts tested: substring(), compareTo(), loops

Key insight: You do NOT need to generate all substrings. The alphabetically largest substring ALWAYS starts with the alphabetically largest character in the string. Why? Because any substring starting with 'z' beats any substring starting with 'a' through 'y'.

Phase 2 — Step-by-step approach (before coding) ▶
1

Loop through each index of the string (i = 0 to length-1)

2

At each index, take s.substring(i) — everything from that position to the end

3

Compare it with the current maximum using .compareTo(). If it's bigger, update the max.

4

Return the max after the loop ends.

Think of it like finding the word that comes LAST in a dictionary. You go through each possible starting position, take everything from there to the end, and keep track of the "latest" one.
Phase 3 — Complete solution with breakdown ▶ 
public static String maxSubstring(String s) {
    String max = "";                        // start with empty string

    for (int i = 0; i < s.length(); i++) {
        String sub = s.substring(i);          // take from index i to end
        if (sub.compareTo(max) > 0) {        // if this comes AFTER max in dictionary
            max = sub;                          // this is the new maximum
        }
    }

    return max;
}

Any time you see "find the maximum/minimum substring alphabetically":
1. Loop through all starting positions
2. Take substring(i) to the end
3. Compare with .compareTo()
4. Track the best one

Time complexity: O(n²) which is fine for length ≤ 100.

Q2: Word Count Tool

You are given a string s that may contain words separated by spaces.

A word is considered valid if it meets ALL of the following:
  • Has at least 3 characters
  • Contains only alphanumeric characters (a-z, A-Z, 0-9)
  • Contains at least one vowel (a, e, i, o, u, case-insensitive)
  • Contains at least one consonant (any letter that is not a vowel)
Your task is to: identify all words, count how many are valid, return that count.

Example 1: s = "This is an example string 234" → Output: 3
WordValid?Reason

anNoLess than 3 characters
exampleYesAll rules pass
stringYesAll rules pass
234NoNo vowel or consonant (only digits)

Example 2: s = "This is Form16 submis$ion date" → Output: 3 (This, Form16, date)
Explain — What is this question actually asking? ▶

In simple words: Break the sentence into words. For each word, check 4 rules. If ALL 4 pass, it's valid. Count the valid ones.

"Alphanumeric" means only letters (a-z, A-Z) and digits (0-9). No special characters like $, !, @, spaces, etc. So "submis$ion" fails because of the $.

"Vowel" = a, e, i, o, u (uppercase counts too). "Consonant" = any letter that is NOT a vowel (b, c, d, f, g...). Digits are neither vowels nor consonants — so "234" has no vowel AND no consonant.

What you need to figure out: How to split a string into words → split(" "). How to check each character → loop through toCharArray() and use Character.isLetterOrDigit().

Phase 1 — Analyze the question type ▶

Type: String splitting + character-by-character validation

Concepts tested: split(), toCharArray(), Character.isLetterOrDigit(), Character.isLetter(), vowel checking

Key insight: You need 4 checks per word. Use boolean flags. Loop through characters ONCE and set all flags in that single pass.

Phase 2 — Step-by-step approach ▶
1

Split the string by space: s.split(" ") → array of words

2

For each word, first check: is length ≥ 3? If no, skip it immediately.

3

Loop through each character of the word. Set 3 flags: allAlphaNum, hasVowel, hasConsonant

4

After the character loop, if ALL flags pass → count++

Phase 3 — Complete solution with breakdown ▶ 
public static int countValidWords(String s) {
    String[] words = s.split(" ");
    int count = 0;
    String vowels = "aeiouAEIOU";

    for (String word : words) {
        if (word.length() < 3) continue;     // skip short words immediately

        boolean valid = true;                // assume all chars are alphanumeric
        boolean hasVowel = false;
        boolean hasConsonant = false;

        for (char c : word.toCharArray()) {
            if (!Character.isLetterOrDigit(c)) {
                valid = false;               // found a bad character like $ or !
                break;                       // no need to check further
            }
            if (Character.isLetter(c)) {       // only check vowel/consonant for LETTERS
                if (vowels.indexOf(c) != -1) hasVowel = true;
                else hasConsonant = true;
            }
        }

        if (valid && hasVowel && hasConsonant) count++;
    }
    return count;
}

Q3: Aggregate Marks

Write a query to get students whose SUM of marks ≥ 500. Show STUDENT_ID and SUM_OF_MARKS. Sort by STUDENT_ID descending.

Table: marks(STUDENT_ID INT, MARKS INT)
Note: A student can have MULTIPLE rows (multiple marks entries).

Sample Data:
STUDENT_IDMARKS
1450
2200
3260
2300
3250
Expected Output: 3 | 510 then 2 | 500
Explain — What is this question actually asking? ▶

In simple words: Look at the table — student 2 appears TWICE (200 and 300). Student 3 appears TWICE (260 and 250). The question wants you to ADD UP all marks for each student, then only show students whose total is 500 or more.

"GROUP BY" is the key concept here. It means "combine all rows with the same STUDENT_ID into one group." Then SUM adds up the marks WITHIN each group.

"Descending order" means biggest first. Student 3 (510) comes before Student 2 (500).

Why you can't use WHERE: WHERE filters INDIVIDUAL rows. But you need to filter GROUPS (the sum of multiple rows). That's what HAVING does — it filters after grouping.

Phase 1 — Analyze the question type ▶

Type: GROUP BY + aggregate function + HAVING filter

Concepts tested: GROUP BY, SUM(), HAVING, ORDER BY DESC, AS alias

Key insight: Same student appears multiple times → you need GROUP BY to combine their rows. Use HAVING (not WHERE) because you're filtering on the SUM which only exists AFTER grouping.

Phase 2 — Think in plain English first ▶
1

"Make piles by STUDENT_ID" → GROUP BY STUDENT_ID

2

"Add up marks in each pile" → SUM(MARKS)

3

"Only keep piles where total ≥ 500" → HAVING SUM(MARKS) >= 500

4

"Sort by student ID, biggest first" → ORDER BY STUDENT_ID DESC

Phase 3 — Complete solution ▶ 
SELECT STUDENT_ID, SUM(MARKS) AS SUM_OF_MARKS
FROM marks
GROUP BY STUDENT_ID
HAVING SUM(MARKS) >= 500
ORDER BY STUDENT_ID DESC;

Phase 4 — Pattern ▶
Whenever you see "for each X, find the total/average/count of Y where total meets condition":
1. SELECT X, AGGREGATE(Y)
2. FROM table
3. GROUP BY X
4. HAVING AGGREGATE(Y) condition
5. ORDER BY ...

WHERE = filter rows BEFORE grouping. HAVING = filter groups AFTER grouping.
Deep Dive — GROUP BY Internals + WHERE vs HAVING ▶

What GROUP BY Actually Does Inside

GROUP BY creates "buckets" — it puts all rows with the same value into one bucket, then collapses each bucket into a single output row.

Step-by-step: How SQL processes GROUP BY

Step 1 — SQL reads all 5 rows from the table:

STUDENT_IDMARKS
1450
2200
3260
2300
3250

Step 2 — GROUP BY creates buckets by STUDENT_ID:

Bucket for STUDENT_ID = 1:  [450]
Bucket for STUDENT_ID = 2:  [200, 300]
Bucket for STUDENT_ID = 3:  [260, 250]

Step 3 — Aggregate function runs on each bucket:

BucketValuesSUM()COUNT()AVG()MIN()MAX()
Student 1[450]4501450450450
Student 2[200, 300]5002250200300
Student 3[260, 250]5102255250260

Step 4 — HAVING filters these groups: Only keep SUM >= 500 → Student 1 removed.

The Rule After GROUP BY

Once you use GROUP BY, every column in SELECT must be either:

  1. The grouped column itself (e.g., STUDENT_ID) — same for all rows in the bucket
  2. An aggregate function (SUM, COUNT, AVG, MIN, MAX) — calculates ONE value from the bucket
This will ERROR:
SELECT STUDENT_ID, MARKS FROM marks GROUP BY STUDENT_ID;

Why? Student 2's bucket has [200, 300]. If you ask for MARKS, which one should SQL show? It can't decide. You MUST use an aggregate function to tell SQL HOW to collapse multiple values into one.

WHERE vs HAVING — The Full Picture

SQL execution order:

FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
SituationUseWhy
Filter individual rows BEFORE groupingWHERERuns at step 2, before groups exist
Filter groups AFTER groupingHAVINGRuns at step 4, after groups are formed
Both neededUse both!WHERE shrinks data first, HAVING filters groups
Example using both WHERE and HAVING together:

SELECT STUDENT_ID, SUM(MARKS) as TOTAL FROM marks WHERE MARKS > 50 GROUP BY STUDENT_ID HAVING TOTAL >= 500 ORDER BY TOTAL DESC;

Step 1: WHERE throws away any individual mark ≤ 50
Step 2: GROUP BY groups the remaining rows by student
Step 3: HAVING only keeps groups where the sum is ≥ 500
Step 4: ORDER BY sorts the result

Q4: Rental Property Cash Flow Analysis

Two tables: properties(id, name) and cash_flows(property_id, amount). Write a query showing:
  • name — property name
  • cash_flows — total number of cash flow records
  • min_cash_flow — minimum amount
  • max_cash_flow — maximum amount
  • avg_cash_flow — average amount, rounded to 2 decimals
Only include properties with average > 10,000. Sort by name ASC. Retain trailing zeros (e.g., 10279.20).
Explain — What is this question actually asking? ▶

In simple words: You have a list of rental properties and their cash flow records. Each property can have MANY cash flow amounts. You need to calculate statistics (count, min, max, average) for each property.

Why it needs a JOIN: Property names are in one table, cash flow amounts are in another. You need to CONNECT them using the property_id foreign key. That's what JOIN does.

"Only include properties with average > 10,000": This is HAVING — filter AFTER grouping. You first calculate the average for each property, then throw away properties where the average is too low.

"Retain trailing zeros": ROUND(10279.2, 2) should give 10279.20, not 10279.2. The ROUND function handles this.

Think of it as: This is Q3 (Aggregate Marks) but with TWO tables instead of one, and MORE aggregate functions in the same query.

Phase 1 — Analyze ▶

Type: JOIN + GROUP BY + multiple aggregates + HAVING + ROUND

Why it's harder: Two tables (need JOIN), five aggregates in one query, ROUND for decimals, HAVING on AVG.

Key insight: This is just Q3 but with a JOIN on top and more aggregate functions. The pattern is identical.

Phase 2 — Build the query piece by piece ▶
1

Connect the tables: FROM properties p JOIN cash_flows c ON p.id = c.property_id

2

Group by property: GROUP BY p.id, p.name

3

Calculate all stats: COUNT, MIN, MAX, ROUND(AVG, 2) — all in one SELECT

4

Filter: HAVING AVG(c.amount) > 10000

5

Sort: ORDER BY p.name ASC

Phase 3 — Complete solution ▶ 
SELECT
    p.name,
    COUNT(c.amount)          AS cash_flows,
    MIN(c.amount)            AS min_cash_flow,
    MAX(c.amount)            AS max_cash_flow,
    ROUND(AVG(c.amount), 2)  AS avg_cash_flow
FROM properties p
JOIN cash_flows c ON p.id = c.property_id
GROUP BY p.id, p.name
HAVING AVG(c.amount) > 10000
ORDER BY p.name ASC;
Common mistakes on this type:
1. Forgetting JOIN → you can't access both tables without it
2. Using WHERE instead of HAVING for the AVG filter
3. Forgetting ROUND → wrong decimal places
4. Forgetting GROUP BY → aggregates won't work per property
Deep Dive — JOIN Types: When to Use Which ▶

Why JOINs Exist

Property NAME is in one table. Cash flow AMOUNT is in another. To get "Cozy Cottage had a cash flow of 3762", you need to connect rows from two tables based on a matching condition. That's what JOIN does.

How JOIN Works Internally

Step-by-step JOIN execution

Step 1: SQL takes every row from properties

Step 2: For each property, SQL looks in cash_flows for rows where property_id matches p.id

Step 3: Every match creates a combined output row:

p.namec.amountHow it matched
Cozy Cottage3762p.id=1 matched c.property_id=1
Cozy Cottage4513p.id=1 matched c.property_id=1 (2nd cash flow)
Sunset Villa2318p.id=2 matched c.property_id=2
Harmony Haven1065p.id=3 matched c.property_id=3

Cozy Cottage appears TWICE because it has two cash flows. JOIN creates one row for every matching combination.

ON vs WHERE for Join Condition

MethodSyntaxVerdict
Using ON (correct)JOIN cash_flows c ON p.id = c.property_idStandard, always use this
Using WHERE (works)JOIN cash_flows c WHERE p.id = c.property_idWorks for INNER JOIN only, avoid

ON says "this is HOW the tables connect." WHERE says "this is a filter." Mixing them is confusing. And for LEFT/RIGHT JOINs, they behave differently.

The 4 Types of JOINs

Imagine a 4th property Mountain Lodge with NO cash flows, and a cash_flow record with property_id=99 that doesn't match any property:

INNER JOIN (or just JOIN)

Rule: Only rows that have a match in BOTH tables.

nameamount
Cozy Cottage3762
Sunset Villa2318
Harmony Haven1065

Mountain Lodge (no cash flows) → EXCLUDED. property_id=99 (no property) → EXCLUDED.

When to use: When you ONLY want data that exists in both tables. "Show me properties that HAVE cash flows." This is the most common JOIN.

LEFT JOIN

Rule: ALL rows from the LEFT table, even if no match. Missing values become NULL.

nameamount
Cozy Cottage3762
Sunset Villa2318
Harmony Haven1065
Mountain LodgeNULL

When to use: When you want ALL records from the main table, even those without matches. "Show ALL properties, even ones with no cash flows."

RIGHT JOIN

Rule: ALL rows from the RIGHT table, even if no match in left.

nameamount
Cozy Cottage3762
Sunset Villa2318
Harmony Haven1065
NULL9999

When to use: Rarely. Most people swap the table order and use LEFT JOIN instead. Same result.

FULL OUTER JOIN

Rule: ALL rows from BOTH tables. NULL where no match.

nameamount
Cozy Cottage3762
Sunset Villa2318
Harmony Haven1065
Mountain LodgeNULL
NULL9999

When to use: When you want EVERYTHING. Very rare. MySQL doesn't support it directly.

Quick decision guide for HackerRank:
1. Does the question say "show ALL X even without Y"? → LEFT JOIN
2. Does the question only want matched data? → INNER JOIN (just JOIN)
3. 99% of HackerRank questions use INNER JOIN.

Why all 3 SQL questions in the exam use INNER JOIN: They only ask for properties/tasks/students that HAVE related data. No question asks "show properties even if they have no cash flows."

Q5: Video Processing — Total Execution Time

Two tables: tasks(id, hash) and processes(task_id, start_dt, end_dt). Calculate total execution time in seconds for each task across all its processes. Return hash and usage_time. Sort by usage_time DESC.
Explain — What is this question actually asking? ▶

In simple words: A video processing system has "tasks." Each task runs multiple "processes." Each process has a start time and an end time. You need to find out: how many TOTAL seconds did each task spend processing?

Example: Task 1 has 3 processes. Process A ran for 619 seconds, Process B for 933 seconds, Process C for 3141 seconds. Total: 619 + 933 + 3141 = 4693 seconds.

The tricky part: How do you calculate "619 seconds" from two timestamps like "2023-04-20 02:01:16" and "2023-04-20 02:11:35"? That's where TIMESTAMPDIFF comes in — it calculates the difference between two dates/times in whatever unit you want.

"hash" is just a unique identifier string for each task. Think of it as the task's name. You need to show it in the output.

Phase 1 — Analyze ▶

Type: JOIN + date arithmetic + SUM + GROUP BY

New concept: TIMESTAMPDIFF (MySQL) or EXTRACT EPOCH (PostgreSQL) to calculate seconds between two timestamps.

Key insight: Each task has MULTIPLE processes. You need to calculate the duration of EACH process (end - start), then SUM them all for each task.

Phase 2 + 3 — Solution (both MySQL and PostgreSQL) ▶ 
-- MySQL:
SELECT t.hash,
    SUM(TIMESTAMPDIFF(SECOND, p.start_dt, p.end_dt)) AS usage_time
FROM tasks t
JOIN processes p ON t.id = p.task_id
GROUP BY t.hash
ORDER BY usage_time DESC;
-- PostgreSQL:
SELECT t.hash,
    SUM(EXTRACT(EPOCH FROM
        (p.end_dt::timestamp - p.start_dt::timestamp)
    ))::int AS usage_time
FROM tasks t
JOIN processes p ON t.id = p.task_id
GROUP BY t.hash
ORDER BY usage_time DESC;
Date/time functions to remember:
MySQL: TIMESTAMPDIFF(SECOND, start, end) — gives difference in seconds
PostgreSQL: EXTRACT(EPOCH FROM (end - start)) — gives difference in seconds

Q6: Energy Bar — Chocolate API

Given a country, call the chocolate API. Find the chocolate with the most energy per gram. Energy = kcal × 0.01 × average_weight. For multiple weights, use average (floored). If tied on energy, pick the one with lowest average price (floored). Return "brand:type".

API: https://jsonmock.hackerrank.com/api/chocolates?countryOfOrigin={country}&page={num}
Explain — What is this question actually asking? ▶

In simple words: There's a database of chocolates on the internet. You access it by visiting a URL. The URL gives you data in JSON format. Your job is to: visit the URL, read the data, do some math, and find the best chocolate.

"Energy per gram": Each chocolate has a kcal value (energy) and a weight. Energy per gram = kcal × 0.01 × weight. Higher = more energy = better. You want the chocolate with the MOST energy per gram.

"Multiple weights": A chocolate might come in different pack sizes [125g, 71g, 116g]. You average them: (125+71+116)/3 = 104. Use integer division (no decimals).

"Pagination": There are too many chocolates to send at once. The API sends them in PAGES (page 1, page 2...). You need to check ALL pages to find the best one.

"Tiebreaker": If two chocolates have the same energy, pick the one with the lower average price.

Phase 1 — Break down the problem ▶

This question stacks 5 concepts:

  1. Call an API from Java (HTTP GET)
  2. Parse JSON response
  3. Handle pagination (loop all pages)
  4. Calculate: average weight, energy per gram
  5. Compare and track the best (with tiebreaker)

Strategy: Use the template from the concepts page. Only change the calculation logic inside the loop.

Phase 2 — The math ▶
Calculation for one chocolate

Weights: [125, 71, 116, 100, 126] → Average = (125+71+116+100+126) / 5 = 107.6 → Floor = 107

kcal: 384

Energy = kcal × 0.01 × avgWeight = 384 × 0.01 × 107 = 410.88 → Floor = 410

Prices: [614, 283, 564, 365, 689] → Average = 503 (for tiebreaker)

// Average weight (integer division = automatic floor)
int totalW = 0;
for (int j = 0; j < weights.size(); j++) totalW += weights.get(j).getAsInt();
int avgWeight = totalW / weights.size();  // 538 / 5 = 107

// Energy per gram (cast to int = floor)
int energy = (int)(kcal * 0.01 * avgWeight);  // 410
Phase 3 — Complete solution ▶

Full Java solution using Gson + HttpClient (what HackerRank actually uses):

public static String energyBar(String country) throws Exception {
    int page = 1, totalPages = 1;
    int maxEnergy = -1, minAvgPrice = Integer.MAX_VALUE;
    String result = "";

    while (page <= totalPages) {
        String url = "https://jsonmock.hackerrank.com/api/chocolates?countryOfOrigin="
                     + country + "&page=" + page;

        // HttpClient — 3-line HTTP call
        String json = HttpClient.newHttpClient()
            .send(HttpRequest.newBuilder().uri(URI.create(url)).build(),
                  HttpResponse.BodyHandlers.ofString()).body();

        JsonObject resp = JsonParser.parseString(json).getAsJsonObject();
        totalPages = resp.get("total_pages").getAsInt();
        JsonArray data = resp.getAsJsonArray("data");

        for (int i = 0; i < data.size(); i++) {
            JsonObject choco = data.get(i).getAsJsonObject();
            int kcal = choco.getAsJsonObject("nutritionalInformation").get("kcal").getAsInt();

            // Average weight (integer division = floor)
            JsonArray weights = choco.getAsJsonArray("weights");
            int sumW = 0;
            for (int j = 0; j < weights.size(); j++) sumW += weights.get(j).getAsInt();
            int avgWeight = sumW / weights.size();

            // Energy per gram
            int energy = (int)(kcal * 0.01 * avgWeight);

            // Average price (for tiebreaker)
            JsonArray prices = choco.getAsJsonArray("prices");
            int sumP = 0;
            for (int j = 0; j < prices.size(); j++) sumP += prices.get(j).getAsInt();
            int avgPrice = sumP / prices.size();

            // Compare: highest energy wins. If tie, lowest price wins.
            if (energy > maxEnergy || (energy == maxEnergy && avgPrice < minAvgPrice)) {
                maxEnergy = energy;
                minAvgPrice = avgPrice;
                result = choco.get("brand").getAsString() + ":" + choco.get("type").getAsString();
            }
        }
        page++;
    }
    return result;
}

Q7: Highest International Students

Given two city names, query a university API. Find the university with the highest number of international students in the first city. If no university exists in the first city, use the second city instead.

API: https://jsonmock.hackerrank.com/api/universities?page={num}

Trap: international_students is a STRING with commas like "15,075", not a number!
Trap: City is inside a nested location object.
Explain — What is this question actually asking? ▶

Q9: Merge Magic — Minimum Power

You are given an array cells of integers. Repeatedly merge any two cells into one (new value = x + y, power used = x + y) until one remains.

Find the minimum total power needed.

Example: cells = [20, 30, 40]
• Merge 20+30=50, power=50. Now [50, 40]
• Merge 50+40=90, power=90. Now [90]
• Total = 50+90 = 140

Sample Input 0: [30, 10, 20] → Output: 90
Sample Input 1: [100, 1] → Output: 101
Explain — What is this question actually asking? ▶

In simple words: You have numbers. Keep picking any two, add them (that costs energy), and put the sum back. Repeat until one number remains. Find the CHEAPEST way to do it.

Why order matters: With [30, 10, 20]:

  • Merge 30+10 first: cost 40, then 40+20=60. Total = 100
  • Merge 10+20 first: cost 30, then 30+30=60. Total = 90 ← cheaper!
  • Merge 30+20 first: cost 50, then 50+10=60. Total = 110

The key insight: Always merging the TWO SMALLEST numbers first gives the minimum cost. Why? Big numbers that merge early get "re-added" in every future merge. Keep the big numbers for last.

This is the Huffman Coding algorithm — a famous "greedy" approach. Greedy means: at each step, make the locally best choice (smallest two), and it guarantees the globally best answer.

Phase 1 — What data structure do we need? ▶

We need to repeatedly: find the two smallest → remove them → add their sum back.

Without PriorityQueue: Sort the array after every merge → O(n log n) each time. For n merges, that's O(n² log n). Slow.

With PriorityQueue (Min-Heap):

  • poll() — removes and returns the SMALLEST element → O(log n)
  • add() — inserts a new element in the right position → O(log n)

So each merge operation is O(log n). For n-1 merges total: O(n log n). Much faster.

Think of PriorityQueue as a magic bag. You throw numbers in, and every time you reach in, it always gives you the smallest one first. You don't need to sort — it maintains order automatically.
Phase 2 — Step-by-step approach ▶
1

Create a PriorityQueue (min-heap) and load all numbers into it

2

While more than 1 element: pull out two smallest, add them, add cost to total, put sum back

3

Return total cost

Phase 3 — Complete solution with word-by-word breakdown ▶ 
public static int minPower(List<Integer> cells) {

    // Create a min-heap. By default, PriorityQueue keeps smallest on top.
    PriorityQueue<Integer> p = new PriorityQueue<>();

    // Load all numbers into the heap
    for (int i = 0; i < cells.size(); i++) {
        p.add(cells.get(i));
    }

    int total = 0;  // running total of all merge costs

    // Keep merging while more than 1 element remains
    while (p.size() > 1) {
        int first = p.poll();   // removes & returns the SMALLEST
        int second = p.poll();  // removes & returns the NEXT smallest
        int sum = first + second;
        total += sum;           // add merge cost to running total
        p.add(sum);             // put merged value BACK into the heap
    }

    return total;
}

Q10: Password Creation — Merge Two Strings

Explain — What is this question actually asking? ▶

In simple words: Interleave two strings like shuffling two decks of cards. Take one from the left pile, one from the right pile, repeat. If one pile runs out, just keep going with the other.

This is a "zipper merge" pattern — walk through both strings at the same time using the same index.

Phase 1 — Why StringBuilder, not String? ▶

In Java, String is immutable — it can NEVER be changed after creation. Every time you do str + "x", Java creates a brand new String object in memory. If you're adding 25,000 characters one by one, that means 25,000 new objects → extremely slow.

StringBuilder is mutable — it modifies itself in place. One object, keeps growing. Much faster.

public static String newPassword(String a, String b) {

    // Mutable string builder — efficient for character-by-character building
    StringBuilder result = new StringBuilder();

    // Loop for as long as the LONGER string
    int max = Math.max(a.length(), b.length());

    for (int i = 0; i < max; i++) {

        // Only add from a if we haven't passed its end
        if (i < a.length()) {
            result.append(a.charAt(i));
        }

        // Only add from b if we haven't passed its end
        if (i < b.length()) {
            result.append(b.charAt(i));
        }
    }

    // Convert back to String (because return type is String)
    return result.toString();
}

Q11: Country Codes — Format Phone Number

Given a country name and a phone number, query the API at:
https://jsonmock.hackerrank.com/api/countries?name=<country>

• If the country is found, get its calling codes. Use the last calling code (highest index).
• Format: +<code> <phoneNumber> (e.g., +93 656445445)
• If country not found (empty data array), return "-1"

Sample 1: country = "Afghanistan", phone = "656445445" → +93 656445445
Sample 2: country = "Puerto Rico", phone = "564593986" → +1939 564593986
(Puerto Rico has callingCodes = ["1787", "1939"] — use last one)
Sample 3: country = "Oceania", phone = "987574876" → -1 (not found)
Explain — What is this question actually asking? ▶

In simple words: Call an API to look up a country. If it exists, get its phone calling code and format a phone number. If it doesn't exist, return "-1".

This is the SIMPLEST REST API question — no pagination needed. One API call, check if data exists, extract a field, format a string.

The trick: A country can have MULTIPLE calling codes (like Puerto Rico with "1787" and "1939"). The question says "use the one at the highest index" — that means the LAST element in the array.

Phase 1 — Analyze ▶

Type: Single API call + JSON parsing + string formatting

No pagination — the response has exactly 0 or 1 records. No while(page <= totalPages) needed.

Key checks:

  1. Is the data array empty? → return "-1"
  2. Get callingCodes array → pick the LAST element
  3. Format as "+" + code + " " + phoneNumber
Phase 2 — Step-by-step approach ▶
1

Build the URL with the country name (replace spaces with %20)

2

Make the HTTP GET request, read the response

3

Parse JSON, check if data array is empty → return "-1"

4

Get the first (and only) object from data. Extract callingCodes array.

5

Pick the LAST element: callingCodes[length - 1]

6

Format and return: "+" + code + " " + phoneNumber

Phase 3 — Complete solution (Gson version for HackerRank) ▶ 
import java.io.*;
import java.net.*;
import com.google.gson.*;

class Result {
    public static String getPhoneNumbers(String country, String phoneNumber) {
        try {
            // 1. Build URL — replace spaces with %20 for URL encoding
            String urlString =
                "https://jsonmock.hackerrank.com/api/countries?name="
                + country.replace(" ", "%20");

            // 2. Make HTTP GET request
            URL url = new URL(urlString);
            HttpURLConnection conn = (HttpURLConnection) url.openConnection();
            conn.setRequestMethod("GET");

            // 3. Read the response
            BufferedReader br = new BufferedReader(
                new InputStreamReader(conn.getInputStream()));
            StringBuilder sb = new StringBuilder();
            String line;
            while ((line = br.readLine()) != null) { sb.append(line); }
            br.close();

            // 4. Parse JSON
            JsonObject response = JsonParser.parseString(sb.toString())
                .getAsJsonObject();
            JsonArray data = response.getAsJsonArray("data");

            // 5. If data is empty → country not found
            if (data.size() == 0) {
                return "-1";
            }

            // 6. Get the country object (first and only element)
            JsonObject countryObj = data.get(0).getAsJsonObject();
            JsonArray callingCodes = countryObj.getAsJsonArray("callingCodes");

            // 7. Pick the LAST calling code (highest index)
            String code = callingCodes
                .get(callingCodes.size() - 1).getAsString();

            // 8. Format and return
            return "+" + code + " " + phoneNumber;

        } catch (Exception e) {
            return "-1";
        }
    }
}

public class Solution {
    public static void main(String[] args) throws IOException {
        BufferedReader bufferedReader = new BufferedReader(
            new InputStreamReader(System.in));
        BufferedWriter bufferedWriter = new BufferedWriter(
            new FileWriter(System.getenv("OUTPUT_PATH")));

        String country = bufferedReader.readLine();
        String phoneNumber = bufferedReader.readLine();
        String result = Result.getPhoneNumbers(country, phoneNumber);

        bufferedWriter.write(result);
        bufferedWriter.newLine();
        bufferedReader.close();
        bufferedWriter.close();
    }
}

Phase 4 — Pattern: No-Pagination API ▶
Not all REST API questions have pagination!

If the API returns 0 or 1 records (like this one), you don't need the while(page <= totalPages) loop. Just:
1. Make ONE API call
2. Check if data is empty → handle the "not found" case
3. Extract what you need from data[0]
4. Format and return

How to know: Read the problem carefully. If it mentions "paginated" or "total_pages", you need the loop. If it says "data contains 0 or 1 records", you don't.
